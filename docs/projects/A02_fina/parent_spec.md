# A02_fina 親仕様（SEO運用OS版）

## ■ 定義

A02_fina は、単なる記事生成ツールではなく、

**検索意図設計 → 構成生成 → ページ生成 → 公開 → 計測 → 改善 → リライト**

までを一貫して扱う **SEO運用OS** である。

## ■ 目的

- コンテンツマーケティングを「人依存」から「システム運用」に変換する
- vertical（業種）を切り替えながら再利用可能な基盤を作る
- AIを活用して設計・生成・改善の大部分を自動化する

## ■ 基本原則

1. 本文生成は禁止（構成・メタまで）
2. 業種ロジックはハードコードせず policy で外出し
3. 1記事単位ではなく「サイト単位」で設計する
4. カニバリ防止を最優先とする
5. AIは生成・補助・自動実行を担い、最終判断は人間

## ■ アーキテクチャ

### 1. 業種非依存コア

- topic_intake
- blueprint_generator
- quality_gate
- candidate_queue

### 2. vertical policy

config/*.json で定義

- 禁止表現
- CTAテンプレ
- リスク区分
- 必須注意文
- YMYL判定

### 3. プラグイン拡張（後日）

- AI検索対応
- SNS配信
- 画像生成
- 被リンク管理

## ■ フェーズ構成

### Phase A：コンテンツMVP（現在）

- session-01：blueprint生成
- session-02：保存 / 履歴
- session-03：再編集 / duplicate
- session-04：一括生成
- session-05：投稿整形
- session-06：vertical切替

### Phase B：AI基盤

- session-07：AI API設定基盤
- session-08：キュー / スケジューラ

### Phase C：SEO運用

- session-09：キーワードマップ / クラスター
- session-10：カニバリチェック / 公開順
- session-11：URL設計
- session-12：記事HTML生成
- session-13：E-E-A-T / 構造化データ
- session-14：sitemap / robots / OGP
- session-15：デプロイ自動化
- session-16：GA4 / Search Console
- session-17：リライトスケジューラ

### 後日プラグイン

- AI検索対応
- SNSマーケティング
- Googleビジネスプロフィール
- 画像生成
- 被リンク戦略

## ■ 実行フロー

1. キーワード入力
2. blueprint生成
3. 保存
4. クラスター設計
5. カニバリチェック
6. URL決定
7. ページ生成
8. 公開
9. 計測
10. リライト

## ■ AIの役割

AIは以下を担う：

- 構成生成
- クラスター候補生成
- URL候補生成
- リライト提案
- 公開順提案

人間は以下を担う：

- 最終承認
- リスク判断
- 法務判断

## ■ 禁止事項

- 本文生成
- 業種ロジックの直接埋め込み
- DB導入（Phase A〜C）
- 認証機能
- スコープ外機能の混入

## ■ 成功条件

- カニバリしない構造で記事生成できる
- 1クリックで量産できる
- 公開→改善サイクルが回る
- vertical切替が可能
- AIを使った自動運用が成立する

## ■ 最重要思想

「記事を作る」のではなく、

**検索構造を設計し、運用し続けるシステムを作る**
