# session-126 drift baseline report

## 結論

既存 session 群の drift 準拠率を実測した。
本 report は default-on 移行判断の材料を提供するものであり、
具体閾値を正本として確定するものではない。

## 実測結果サマリ

- 対象 session 数: 91
- 対象 acceptance 数: 81

### 観点別準拠率

| 観点 | 準拠 | 非準拠 | 準拠率 |
|---|---|---|---|
| required keys | 40 | 51 | 43.96% |
| review_points 正本準拠 | 4 | 87 | 4.40% |
| allowed_changes_detail | 53 | 38 | 58.24% |
| acceptance.test_name | 63 | 18 | 77.78% |

### review_points 第4軸の分布

- 正本「検証十分性」: 4 件
- legacy「実装過不足なし」: 31 件
- その他 variant: 56 件

## 実測から導ける解釈

- **default-on に近い状態か**: 正本どおりの `review_points`（4 軸完全一致ではなく第4軸のみで見ても）が 91 件中 4 件にとどまり、必須 13 キー充足は約 44% のため、現状の定義群だけを見ると default-on に切り替えるには材料不足であり、一定規模の定義整理または新規 session への段階的強制が前提になりうる。

- **新規のみ強制で足りるか / 既存整理が要るか**: `allowed_changes_detail` や `acceptance.test_name` の方が相対的に準拠率は高い一方、必須キーと第4軸の乖離が大きい。ブロック系・旧 phase2 系・多数の session-8x 系に欠落が集中しており、**新規だけ整備しても既存の底上げがなければ全体分布は大きく動かない**可能性が高い。

- **drift detector 4部作（session-123 / 124 / 125 / 125a）の整合性**: session-123 は第4軸が legacy 表現（「実装過不足なし」）のまま残っており、session-124・session-125・session-125a は第4軸が正本「検証十分性」に一致する。ポリシー文書を手掛けたセッション群は意図どおり正本に寄せられている一方、先行する session-123 は歴史的文言が残っている。

## 不確実な点

- 実測基準の網羅性（本調査は drift_detector v0.1 と同一の規則に合わせているが、運用上の追加チェックがある場合は別カウントになりうる）

- 一部 session の歴史的例外として残すべきか、是正対象かのポリシー判断（特にブロック／運用実験系ファイル）

- legacy の第4軸が意図的に残っているものか、単なる更新漏れかは、この実測だけでは判別できない
